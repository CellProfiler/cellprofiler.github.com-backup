Documentation

The Developers' wiki orients developers to the structure of CellProfiler's code, the details of how modules function, the proper use of settings and their display in the user interface, and how image and object measurements are processed and stored.

What if I want to contribute to CellProfiler?

We will be glad to take a look at any code you wish to contribute to the CellProfiler Python project, though we cannot guarantee that the code will be included. Please refer to this section of the developer wiki.

If you are contributing code, keep these considerations in mind:

  • If possible, include your code as an extension to a current module, such as by adding a new setting.
  • If your code is not suited for any of the current modules, create a new module with the new functionality.
We also recommend the following:
  • During implementation, try to mimic the style already present in the CellProfiler project source code. See our style guide for more details.
  • Provide test code and test cases. The new functionality should pass current and new tests or give expected results. See the section on tests for more details.
  • If possible, test the code on more than one platform. It is good to combine testing on Windows with testing on Linux, Mac OS X or another Unix platform.

SVN Repositories

The source code for the CellProfiler project is hosted in a web-browsable subversion (SVN) repository administered by the Broad Institute, which can be checked out at these locations:

  • CellProfiler 2.0 (Python 2.5): https://svn.broadinstitute.org/CellProfiler/trunk/CellProfiler/
  • CellProfiler Analyst 2.0 (Python 2.5): https://svn.broadinstitute.org/CellProfiler/trunk/CPAnalyst/
  •  
  • Deprecated versions, no longer actively developed:
  • CellProfiler 1.0 (MATLAB, no longer under development): https://svn.broadinstitute.org/CellProfiler/trunk/CellProfiler_old_matlab/
  • CellProfiler Analyst 1.0 (Java 1.6, no longer under development): https://svn.broadinstitute.org/CellProfiler/trunk/oldCPA/

Trunk Builds

Compiled, bleeding-edge versions of CellProfiler are available as trunk builds.

Bugs and Feature Requests

If you have encountered a bug, please contact us through the forum. Our bug tracking is currently not visible to users outside the Broad Institute.

Mailing Lists

There are development lists for the CellProfiler (archive) and CellProfiler Analyst tools (archive). These are not high-traffic lists and are intended for discussions or questions ranging from code details to long-term plans for the projects.To subscribe, send mail with "Subscribe" in the Subject field to:

  • cellprofiler-dev-request@broadinstitute.org
  • cpa-dev-request@broadinstitute.org
